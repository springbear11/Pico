# PicoATE 变量与结果引用

## 两个解析阶段

### 配置期变量

配置期变量在加载 Sequence、Station config 和 NativeHost manifest 时解析，常用变量包括：

```text
${PROJECT_DIR}
${SEQUENCE_DIR}
${PICOATE_NATIVE_HOST}
```

配置期解析由 `VariableResolver` 完成，主要用于路径、程序位置和环境配置。

### 运行期变量

运行期变量在每个 Node Attempt 执行前解析：

```text
${uut.id}
${frame.id}
${attempt.index}
${attempt.number}
${loop.index}
${loop.value}
${var.serialNumber}
```

完整字符串表达式保留原始 QVariant 类型；嵌入普通字符串时转换为文本。

## 作用域 Step 地址

顶层 TestItem 和独立 Step 可以使用三位字符串 ID：

```json
{ "id": "001", "name": "CAN请求" }
```

TestItem 内子步骤可以同时使用局部 ID 和语义 Key：

```json
{ "id": "03", "key": "rx", "name": "接收响应" }
```

编译后节点路径：

```text
001.rx
```

规则：

- ID 必须是 JSON 字符串，`"001"` 不能写成数字 `1`。
- 顶层节点 ID 在整个 Sequence 中唯一。
- 子步骤 ID 和 Key 只需在同一个父节点下唯一。
- ID/Key 创建后不因插入、删除或排序而自动重编号。
- ID/Key 不允许包含 `.`。
- `outputs`、`measurements`、`outcome` 是保留 Key。
- 执行顺序由数组和图边决定，与数字大小无关。

## Step 结果表达式

跨 TestItem 使用完整路径：

```json
"frame": "${step:001.rx.outputs.frame}"
```

当前 TestItem 内使用相对 Key：

```json
"actual": "${step:parse.outputs.voltage}"
```

读取结果状态：

```json
"rxOutcome": "${step:001.rx.outcome}"
```

读取测量结果：

```json
"firstMeasurement": "${step:001.rx.measurements.0}"
```

## ExecutionResultStore

每个 `ExecutionSession` 持有一个 `ExecutionResultStore`，存储键包含：

```text
UutId + FrameId + NodePath
```

因此两个 UUT 同时运行 `001.rx` 时，读取到的始终是各自结果。

仓库行为：

- 普通 Step 保存最终 NodeResult。
- Retry 保存每个 Attempt 历史，默认引用最新结果。
- Loop body 保存每轮结果，Loop 结束后默认引用最后一轮。
- `outputs` 和 `measurements` 默认只允许读取 Passed 源结果。
- `outcome` 可以读取任意已完成源结果。
- 大型波形和抓包文件应保存为 Artifact，outputs 中只返回路径和摘要。

## 编译期失效检查

编译器会拒绝：

- 节点不存在
- 表达式格式错误
- 自引用
- 引用未来节点
- 无法保证源节点先执行
- 同级 ID/Key 重复
- Key 使用保留字段

有效引用会生成 `condition = step-result` 的数据依赖边。

## 运行期失效检查

运行期可能返回：

| 错误码 | 含义 |
|---|---|
| `StepReferenceNotFound` | 节点地址不存在 |
| `StepResultUnavailable` | 源节点尚无结果 |
| `StepResultNotPassed` | 源节点 Failed/Error/Timeout/Skipped |
| `StepResultValueNotFound` | outputs/measurements 中不存在目标字段 |
| `RuntimeVariableResolutionError` | NodeRunner 汇总的变量解析错误 |

完整示例见 `examples/scoped_result_sequence.json`。
