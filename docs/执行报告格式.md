# PicoATE ExecutionReport 存储格式

版本：V3  
日期：2026-07-01

## 1. 用途

`ExecutionReport` JSON 是测试结束后的只读结果文件，用于 UI 历史查看、离线分析、
JSON/CSV 导出和后续 MES/数据库接入。它不是运行会话快照，不能用于断点恢复，也
不包含正在运行的 `ExecutionSession`、设备连接或资源租约。

## 2. 顶层格式

```json
{
  "schema": "picoate.execution-report",
  "schemaVersion": 3,
  "report": {
    "planId": "...",
    "sequenceId": "...",
    "sequenceVersion": "...",
    "state": "Completed",
    "completed": true,
    "hasError": false,
    "uuts": []
  }
}
```

`schemaVersion` 是文件兼容边界。当前读取器接受 V1、V2 和 V3；V2 新增
`StepReport.children` 保存复合测试项层级。遇到未知更高版本时明确报错，不会猜测
字段含义。缺少可选数组或可选属性时使用空值。

V3 新增 `StepReport.nodePath`。`stepId` 保存当前作用域内的局部 ID，例如 `03`；
`nodePath` 保存编译后的唯一地址，例如 `001.rx`。UI 默认显示局部 ID，历史对账、
RuntimeEvent 和结果仓库使用 `nodePath`。

## 3. 数据层级

```text
ExecutionReport
  UutReport[]
    StepReport[]
      children[]             # testItem/loop child steps
      AttemptReport[]
        LoopIterationContext
        MeasurementResult[]
      MeasurementResult[]
```

Measurement 保存 value、rawValue、unit、上下限、判定、错误和 attributes。JSON
加载后不重新计算 Pass/Fail；文件中的报告结果就是历史事实。

## 4. 历史目录

Runner 默认保存到 Qt `AppLocalDataLocation/history`：

```text
history/
  index.json
  20260629-203000-123-a1b2c3d4.json
```

报告和索引均使用 `QSaveFile` 原子替换。`index.json` 损坏或丢失时，Runner 会扫描
目录内有效的 ExecutionReport 文件重建索引；无法解析的损坏报告不会进入历史列表。

## 5. CSV 导出

CSV 使用 UTF-8 BOM 和 CRLF，方便 Excel 打开中文。每行对应一个 Measurement；没有
Measurement 时仍输出 Attempt 或 Step 行。字段包括：UUT、Step、Step Outcome、
Attempt、Loop iteration/value、Measurement、value/unit/limits/status 和错误信息。

## 6. 明确不包含

- Checkpoint/Session 恢复状态；
- DLL 内部实时日志；
- RuntimeEvent 时间线；
- 设备连接句柄、Host 进程和资源租约；
- UI 布局或筛选条件。
